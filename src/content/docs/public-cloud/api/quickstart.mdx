## API Quickstart

Authenticate and make your first API calls.

### 1. Get your token

Generate a Bearer token from the portal under **Profile → API Tokens**. Every request uses the base
URL `https://api.zcp.zsoftly.ca/api` and sends the token in the `Authorization` header.

### 2. List your virtual machines

The example below authenticates and lists your virtual machines. Pick your language and replace
`your-token` with your API token.

<Tabs syncKey="lang">
<TabItem label="cURL">

```bash
curl -s "https://api.zcp.zsoftly.ca/api/virtual-machines" \
  -H "Authorization: Bearer your-token" \
  -H "Accept: application/json" | jq '.data'
```

</TabItem>
<TabItem label="Python">

```python
import requests

BASE = "https://api.zcp.zsoftly.ca/api"
TOKEN = "your-token"  # Profile -> API Tokens

resp = requests.get(
    f"{BASE}/virtual-machines",
    headers={"Authorization": f"Bearer {TOKEN}", "Accept": "application/json"},
)
for vm in resp.json()["data"]:
    print(vm["name"], vm["state"])
```

```go
package main

import (
	"encoding/json"
	"fmt"
	"net/http"
)

func main() {
	const base = "https://api.zcp.zsoftly.ca/api"
	const token = "your-token" // Profile -> API Tokens

	req, _ := http.NewRequest("GET", base+"/virtual-machines", nil)
	req.Header.Set("Authorization", "Bearer "+token)
	req.Header.Set("Accept", "application/json")

	resp, err := http.DefaultClient.Do(req)
	if err != nil {
		panic(err)
	}
	defer resp.Body.Close()

	var env struct {
		Data []struct {
			Name  string `json:"name"`
			State string `json:"state"`
		} `json:"data"`
	}
	json.NewDecoder(resp.Body).Decode(&env)
	for _, vm := range env.Data {
		fmt.Println(vm.Name, vm.State)
	}
}
```

### 3. Create a virtual machine

Slugs for `project`, `region`, `template`, `plan`, and `networks` come from the corresponding list
endpoints (`/projects`, `/regions`, `/templates`, `/plans/service/compute`, `/networks`). See the
[Cloud Platform API reference](https://api.zcp.zsoftly.ca/api/docs/nimbo) for the full request
schema.

<Tabs syncKey="lang">
<TabItem label="cURL">

```bash
curl -s -X POST "https://api.zcp.zsoftly.ca/api/virtual-machines" \
  -H "Authorization: Bearer your-token" \
  -H "Content-Type: application/json" \
  -H "Accept: application/json" \
  -d '{
    "name": "my-server",
    "project": "my-project",
    "region": "yow",
    "boot_source": "template",
    "template": "ubuntu-24-04",
    "plan": "general-1vcpu-1gb",
    "network_type": "public",
    "networks": ["my-public-network"],
    "is_public": true,
    "billing_cycle": "hourly",
    "hostname": "my-server"
  }' | jq '.data'
```

</TabItem>

### 4. Manage a virtual machine

VM lifecycle actions use the VM slug and the action name, one of `start`, `stop`, `reboot`, or
`reset`.

<Tabs syncKey="lang">
<TabItem label="cURL">

```bash
curl -s -X PUT "https://api.zcp.zsoftly.ca/api/virtual-machines/my-server/stop" \
  -H "Authorization: Bearer your-token" \
  -H "Accept: application/json" | jq '.message'
```
